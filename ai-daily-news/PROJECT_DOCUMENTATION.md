# AI每日资讯日报推送系统 - 完整项目文档

## 🎯 项目概述

### 项目目标
创建一个为非专业人士打造的AI资讯日报推送系统，解决现有AI资讯"太硬"或"太浅"的问题。

### 核心价值
- **受众定位**：完全没有技术背景的普通人（餐饮老板、中学老师、退休人员等）
- **核心痛点**：想了解AI但每次都被术语劝退，形成"AI跟我没关系"的认知脱节
- **成功标准**：读完日报后，普通人能用自己的话转述给家人，且至少有一条新闻让他们觉得"这件事跟我有关系"

---

## 🏗️ 系统架构

### 三层处理架构

```
📡 采集层 → 获取AI资讯素材
🧠 消化层 → 筛选Top5+结构化
💬 说人话层 → 通俗化改写
```

### 技术栈

**后端**：
- Python 3.8+
- Flask (Web框架)
- requests (HTTP请求)

**前端**：
- HTML + CSS + JavaScript
- 响应式设计

**AI服务**：
- NewsAPI (真实新闻采集)
- 老张代理API (Qwen-Plus处理)

## 🛠️ 已知问题与解决方案

### 问题1：URL有效性（部分）

**问题**：部分URL返回403/401状态码
**原因**：部分网站对爬虫访问有限制
**解决**：用户手动点击时成功率会更高，浏览器访问不会被限制

### 问题2：国内新闻相关性

**问题**：国内新闻相关性30%（低于国际的80%）
**原因**：中文新闻源相对较少，关键词匹配需要更精确
**解决**：持续优化中文搜索关键词

### 问题3：API调用费用

**问题**：使用多个API服务可能产生费用
**解决**：
- NewsAPI免费版：每天100次请求（足够使用）
- 老张代理：按使用量计费，需要监控
